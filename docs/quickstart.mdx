title: Quickstart
description: Run the platform locally, create a voice agent, and test a turn.

This guide gets the current OpenDot platform, database, API, and local voice runtime running.

## Run with Docker Compose

From the repository root:

```bash
cp .env.example .env
docker compose up --build
```

Add provider keys to the root `.env` before testing live voice sessions.

Open:

```text
http://localhost:5173
```

Create a local account on the auth page, then open **Agent Studio**.

The Compose stack starts PostgreSQL, applies migrations, serves the web console, and exposes the voice runtime at `http://localhost:8787`. The default local run uses OpenDot email/password auth; Supabase Auth can be enabled later with `SUPABASE_URL`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, and a Supabase `POSTGRES_URI`.

## Inspect the database

Start the local database and migration job if the full stack is not already running:

```bash
docker compose up -d postgres migrate
```

Then open Drizzle Studio:

```bash
cd platform
npm run db:studio
```

Open `https://local.drizzle.studio` to inspect the PostgreSQL schema and rows, including `app_users`, `local_auth_credentials`, `user_preferences`, API keys, versioned agents/pipelines, devices, device state, and deployments.

## Run with npm

From the platform app:

```bash
cd platform
npm install
npm run db:migrate
```

Start the API and web console in separate terminals:

```bash
# terminal 1
npm run api

# terminal 2
npm run dev
```

Open the Vite URL printed in the terminal. It is usually:

```text
http://localhost:5173
```

## Start the voice runtime for npm runs

The browser test needs the voice runtime in another terminal:

```bash
cd platform
npm run runtime
```

Add real provider keys to the root `.env` before using the runtime:

```bash
DEEPGRAM_API_KEY=...
OPENAI_API_KEY=...
```

## Create and test an agent

1. Open **Agent Studio**.
2. Create a draft agent with a name and description.
3. Open **Configuration** and review the default pipeline.
4. Open **Browser Test**.
5. Click **Connect**.
6. Click **Start mic** and speak.
7. Stop speaking and wait for the assistant text and audio chunks.

The default pipeline is:

```text
Deepgram VAD -> Deepgram STT -> OpenAI LLM -> Deepgram TTS
```

## Optional device check

Open **Dot Device** to pair a device, refresh runtime availability, and bind the selected agent config to the device.

The default runtime endpoints are:

```text
WebSocket: ws://localhost:8787/voice
HTTP:      http://localhost:8787
```

## Deploy on Render

The root `render.yaml` Blueprint deploys the same platform shape to Render:
static web, platform API, voice runtime, and API pre-deploy migrations. Use
Supabase Postgres for `POSTGRES_URI`, keep `POSTGRES_SSL=true`, and provide the
public API/runtime URLs for the `VITE_*` build variables in Render. See
[Render deployment](/render-deployment) for the full variable list.
