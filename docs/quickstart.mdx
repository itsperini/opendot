title: Quickstart
description: Run the platform locally, create a voice agent, and test a turn.
---

This guide gets the current OpenDot platform, database, API, and local voice runtime running.

## Run with Docker Compose

From the repository root:

```bash
cp .env.docker.example .env.docker
docker compose --env-file .env.docker up --build
```

Add provider keys to `.env.docker` before testing live voice sessions.

Open:

```text
http://localhost:5173
```

The Compose stack starts PostgreSQL, applies migrations, serves the web console, and exposes the voice runtime at `http://localhost:8787`.

## Inspect the database

Start the local database and migration job if the full stack is not already running:

```bash
docker compose --env-file .env.docker.example up -d postgres migrate
```

Then open Drizzle Studio:

```bash
cd platform
npm run db:studio
```

Open `https://local.drizzle.studio` to inspect the PostgreSQL schema and rows.

## Run with npm

From the platform app:

```bash
cd platform
npm install
npm run db:migrate
```

Start the API and web console in separate terminals:

```bash
# terminal 1
npm run api

# terminal 2
npm run dev
```

Open the Vite URL printed in the terminal. It is usually:

```text
http://localhost:5173
```

## Start the voice runtime for npm runs

The browser test needs the voice runtime in another terminal:

```bash
cd platform
cp .env.example .env
npm run runtime
```

Add real provider keys to `.env` before using the runtime:

```bash
DEEPGRAM_API_KEY=...
OPENAI_API_KEY=...
```

## Create and test an agent

1. Open **Agent Studio**.
2. Create a draft agent with a name and description.
3. Open **Configuration** and review the default pipeline.
4. Open **Browser Test**.
5. Click **Connect**.
6. Click **Start mic** and speak.
7. Stop speaking and wait for the assistant text and audio chunks.

The default pipeline is:

```text
Deepgram VAD -> Deepgram STT -> OpenAI LLM -> Deepgram TTS
```

## Optional device check

Open **Dot Device** to pair a device, refresh runtime availability, and bind the selected agent config to the device.

The default runtime endpoints are:

```text
WebSocket: ws://localhost:8787/voice
HTTP:      http://localhost:8787
```
