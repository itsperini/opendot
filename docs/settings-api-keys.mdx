---
title: Settings and API keys
description: Manage workspace preferences and prototype SDK keys.
---

Settings holds user-level preferences and prototype SDK keys for OpenDot. In the database, profile state is split across `app_users`, `organizations`, and `user_preferences` so Supabase Auth can provide identity without owning OpenDot product authorization.

## Workspace preferences

The profile panel stores:

- **Name**
- **Email**
- **Workspace**
- **Timezone**
- **Compact density**

Compact density tightens page spacing and panel padding across the platform console.

## API keys

Use **API keys** to create and revoke local prototype keys. Active keys expose a prefix in the `OPENDOT_API_KEY` row.

The current prototype stores organization-scoped API key hashes and metadata in PostgreSQL. The full token is only shown when a key is created, so copy it immediately if you need to use it outside the UI. Treat these as local development credentials, not production secrets.

## Copy and revoke

Each key row supports:

- **Copy**: copies the full token to the clipboard.
- **Revoke**: marks the key as revoked and disables copy.

Revoked keys remain visible so you can see when they were created and confirm they are no longer active.
