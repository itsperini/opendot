---
title: Agent Studio
description: Create agents and prepare their runtime configuration.
---

Agent Studio is the first step in the OpenDot flow. It creates draft voice agents that become the anchor for pipeline tuning, model settings, knowledge configuration, live testing, and hardware binding.

## Create a draft agent

Each agent starts with:

- **Agent name**: a short label such as `Lobby guide`
- **Description**: the job the agent should perform

When you create the agent, OpenDot attaches the default four-stage pipeline:

```text
VAD -> STT -> LLM -> TTS
```

The new agent becomes the selected agent for **Configuration**, **Browser Test**, and **Dot Device**. Today the prototype stores the basic draft locally; the agent model is designed to grow into prompts, knowledge, model choices, and deployment bindings.

## Select an existing agent

Use the agent list to switch between drafts. Selecting an agent changes which pipeline is edited, which agent is sent to the runtime, and which config can be bound to a device.

## Prototype storage

The current prototype persists agents locally in the browser. Clearing site data resets the draft list.
